## Две причины недосчёта на $3.05

### Причина 1: M11 slug mismatch (-$2.32, ~76% недосчёта)

В config'е использовался дата-версионированный alias `anthropic/claude-4.8-opus-20260528`, которого нет в `GET /api/v1/models` catalog OpenRouter (хотя `POST /chat/completions` его принимал и реально вызывал Opus 4.8). `calculate_cost()` смотрел в catalog metadata, не находил - возвращал $0.

**Доказательство, что модель реально Opus 4.8** (из audit-файла `phase1_freeresponse_M11_01.json`):

```json
"raw_response_full": {
  "id": "gen-1779994047-qHuh7N4jLQuq4iimVNFq",
  "model": "anthropic/claude-4.8-opus-20260528",
  "provider": "Anthropic",
  ...
  "usage": {
    "prompt_tokens": 568,
    "completion_tokens": 730,
    "cost": 0.02109   ← OpenRouter сам посчитал, наш скрипт это поле игнорировал
  }
}
```

### Причина 2: Reasoning tokens не учитывались (-$0.73, ~24% недосчёта)

Наш `calculate_cost()` умножал только `prompt_tokens × prompt_price + completion_tokens × completion_price`. Но у моделей с reasoning (DeepSeek v4-pro, Kimi k2.6, Llama 4 Maverick) OpenRouter биллит дополнительно за reasoning tokens. Эта стоимость уже была включена в `usage.cost`, но мы её не подтягивали.

Расхождение по моделям с reasoning:
- M4 DeepSeek v4-pro: +$0.56 (reasoning tokens 318 157 за фазу)
- M5 Kimi k2.6: +$0.16
- M9 Llama 4 Maverick: +$0.01

Модели без reasoning (M1, M2, M6, M7, M8, M10) - расхождение ровно $0. Это **очень сильный signal**, что причина именно в reasoning.

## Что починено

1. **`config_phase2j.yaml`**: M11 slug заменён на canonical `anthropic/claude-opus-4.8`.

2. **`run_experiment.py:calculate_cost()`**: переписан с правильной иерархией источников:
   - Шаг 1: `usage.cost` - готовое значение от OpenRouter (с учётом reasoning, cached, audio tokens и применённого тарифа провайдера)
   - Шаг 2: `usage.cost_details.upstream_inference_cost` - запасной breakdown
   - Шаг 3: manual calculation из catalog - крайний случай, с громким warning при $0
   
   Это закрывает оба класса бага: slug mismatch (шаг 1 не зависит от catalog) и reasoning undercount (шаг 1 уже всё учёл).

3. **REPORT.md**: обновлены финансовые числа. Real cost: $15.32, не $12.27 и не $18.96.

## Уроки для будущих фаз

1. **Никогда не пересчитывать стоимость, если API уже её вернул.** Всегда брать `usage.cost`. Только если поля нет - fallback на manual.

2. **Аудит идти от audit-файлов, не от CSV**, если на учётке есть другая активность. Audit-файлы привязаны к конкретному эксперименту и timestamp'у.

3. **Проверять `model_id_used` vs `request_payload.model`** в audit-файлах - убеждаться, что вызвалась нужная модель, а не silent fallback.

4. **Для следующих фаз с новыми моделями**: после первой ячейки сразу запускать audit-скрипт на эту ячейку, ловить $0 cost при ненулевых токенах. Не ждать конца 12-часового прогона.
