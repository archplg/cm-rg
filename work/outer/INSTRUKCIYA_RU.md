# Инструкция: как запустить cross-model эксперимент

Полный путь от пустой машины до готовых результатов. Рассчитано на ~1 час твоего активного времени + ~1 час пассивного ожидания прогона.

---

## Часть 1 - что подготовить заранее (один раз, ~15 минут)

### 1.1 Деньги

Понадобится **$50 USD** на оплату API-вызовов через OpenRouter. Бюджет эксперимента - ~$45, остаток $5 как запас. Кредиты не сгорают, остаток можно использовать в будущем.

Нужна карта Visa/MasterCard, которой можно оплатить покупку в долларах. Российские карты могут не подойти - проверь свою.

### 1.2 Аккаунт OpenRouter

OpenRouter - это единая точка входа к 300+ моделям ИИ (Claude, GPT, Gemini, DeepSeek, Kimi и десятки других). Один API-ключ работает со всеми. Это критично для нашего эксперимента, иначе пришлось бы заводить 5 разных аккаунтов.

Что сделать:
1. Открыть https://openrouter.ai/
2. Нажать "Sign up", войти через Google или GitHub (так быстрее)
3. Перейти в https://openrouter.ai/settings/credits
4. Нажать "Add credits", положить **$50 USD** картой
5. Перейти в https://openrouter.ai/settings/keys
6. Нажать "Create new key", дать ему имя `archipelago-experiment`
7. **СРАЗУ скопировать ключ** (начинается с `sk-or-v1-...`) и сохранить в надёжное место. Повторно увидеть его нельзя - только пересоздать.

### 1.3 Python на компьютере

Если на машине уже есть Python 3.11 или новее - пропусти этот пункт. Проверить можно командой `python --version` в PowerShell.

Если Python не установлен:
1. Открыть https://www.python.org/downloads/
2. Скачать Python 3.12 для Windows (64-bit)
3. Запустить установщик
4. **Важно!** На первом экране поставить галочку "Add python.exe to PATH". Без этой галочки потом ничего работать не будет.
5. Нажать "Install Now", дождаться окончания
6. Открыть PowerShell, проверить: `python --version` - должно показать `Python 3.12.x`

## Часть 2 - установка эксперимента (один раз, ~10 минут)

### 2.1 Распаковать архив

Распаковать `archipelago_cross_model.zip` в `C:\Users\Sergey\archipelago_cross_model\` (используй наш симлинк-путь, не кириллический).

Внутри должно быть:
- `README.md`, `PROTOCOL.md`, `SETUP.md` - документация
- `config.yaml` - настройки эксперимента
- `requirements.txt` - список Python-зависимостей
- `run_experiment.py` - главный скрипт оркестрации
- `analyze.py` - скрипт анализа результатов
- папки `tasks/`, `personas/`, `results/`, `logs/`, `analysis/`

### 2.2 Открыть PowerShell в этой папке

1. Открыть проводник, найти папку `C:\Users\Sergey\archipelago_cross_model\`
2. В адресной строке проводника написать `powershell` и нажать Enter

ИЛИ через меню Пуск:
1. Открыть PowerShell (клавиша Windows, набрать "powershell", Enter)
2. Перейти в папку: `cd C:\Users\Sergey\archipelago_cross_model`

### 2.3 Установить Python-зависимости

В PowerShell внутри папки эксперимента:
```
pip install -r requirements.txt
```

Это скачает: requests, pyyaml, pandas, numpy, scikit-learn, scipy, matplotlib, seaborn. Займёт 3-7 минут. Если увидишь warning'и о версиях кэша - игнорируй; ошибки останавливают установку, warning'и не останавливают.

Если pip ругается, что не находится - значит на шаге 1.3 не поставил галочку PATH. Тогда: переустановить Python и не забыть галочку.

### 2.4 Положить API-ключ в переменную окружения

В том же окне PowerShell (вставь свой ключ вместо `sk-or-v1-...`):

```
$env:OPENROUTER_API_KEY = "sk-or-v1-ТВОЙ-КЛЮЧ-СЮДА"
```

Этот способ положит ключ только в текущее окно PowerShell. Закроешь окно - ключ исчезнет, надо будет вписать заново.

Чтобы сохранить ключ навсегда (только для своего пользователя):
```
[Environment]::SetEnvironmentVariable("OPENROUTER_API_KEY", "sk-or-v1-ТВОЙ-КЛЮЧ-СЮДА", "User")
```

После постоянной установки нужно **закрыть и заново открыть** PowerShell.

**Важно:** ключ нигде не записывать в файлы (включая конфиги в репозитории). Только в переменную окружения.

## Часть 3 - сухой прогон (обязательно перед реальным, ~2 минуты)

Прежде чем тратить деньги, обязательно прогнать в режиме `--dry-run`. Скрипт:
- проверит, что API-ключ работает
- проверит, что все 5 моделей доступны на OpenRouter сегодня
- покажет актуальные цены по каждой модели
- посчитает общую смету эксперимента
- НЕ потратит ни одного цента

В PowerShell:
```
python run_experiment.py --dry-run
```

Что ты должен увидеть:
- `INFO: Pre-flight checks`
- `INFO: 300+ models available on OpenRouter` (или похожее число)
- Строки про каждую из 5 моделей с указанием цен
- `INFO: Estimated total cost: $XX.XX`
- `INFO: DRY RUN complete - no API spend.`

**Если увидишь предупреждения** вида `M2 (openai/gpt-5.5) NOT FOUND in OpenRouter catalog` - это значит, что slug модели за последние недели изменился. Что делать:
1. Открыть https://openrouter.ai/models в браузере
2. Найти текущий slug для нужной модели (по поисковой строке)
3. Открыть `config.yaml` в Блокноте (или любом текстовом редакторе)
4. Заменить устаревший `id:` на актуальный, сохранить
5. Прогнать `--dry-run` ещё раз

Если оценка стоимости вышла выше $80 - значит модели подорожали или мой расчёт не сошёлся. Перепроверь, при необходимости подними `total_budget_usd:` в `config.yaml`.

## Часть 4 - реальный прогон (~30-90 минут)

Когда dry-run прошёл чисто:

```
python run_experiment.py
```

Скрипт ещё раз сделает pre-flight, потом спросит подтверждение:
```
Proceed with the full experiment? Type YES to confirm:
```

Ввести `YES` (заглавными, точно). Любой другой ввод отменит запуск.

Дальше скрипт пойдёт по 10 ячейкам последовательно. Каждая ячейка это: один (задача × условие × повтор), и в каждой - 5 моделей делают 3 фазы (свободный ответ, конструкты, рейтинги). В консоль выводится прогресс:
```
[A_N_run1] Phase 1: free response
  M1 (anthropic/claude-opus-4.7) ok, $0.0234
  M2 (openai/gpt-5.5) ok, $0.0187
  ...
[A_N_run1] Phase 2: anonymize
[A_N_run1] Phase 3: triadic construct elicitation
  M1 ok, 3 constructs, $0.0091
  ...
```

Можно оставить окно открытым и заниматься чем угодно. Каждое окончание ячейки логируется как `[CELL_ID] done. cell cost: $X.XX. total: $YY.YY`.

### Если что-то прерывает прогон

- Закрыл окно случайно, упал интернет, отключилось питание - не страшно
- Промежуточное состояние сохраняется после **каждого API-вызова** в `results/state.json`
- Возобновить:
```
python run_experiment.py --resume
```
Уже выполненные ячейки пропустит, продолжит с того места, где остановилось.

### Чтобы прогнать одну конкретную ячейку

```
python run_experiment.py --cell A_N_run1
```

Список доступных ячеек видно после dry-run: `A_N_run1`, `A_P_run1`, `B_N_run1`, `B_P_run1`, `B_N_run2`, `B_N_run3`, `B_P_run2`, `B_P_run3`, `C_N_run1`, `C_P_run1`.

### Мониторинг расхода в процессе

В отдельном окне PowerShell:
```
type C:\Users\Sergey\archipelago_cross_model\results\state.json | findstr total_cost_usd
```

Покажет текущий потраченный $.

## Часть 5 - анализ результатов (~2 минуты)

Когда прогон закончится (увидишь `All cells processed. Final cost: $XX.XX`):

```
python analyze.py
```

Скрипт:
- Соберёт данные из всех 10 ячеек
- Сделает PCA, кластеризацию, корреляции, метрики согласия
- Построит биплоты и тепловые карты для каждой ячейки
- Оценит гипотезы по pre-registered критериям из PROTOCOL.md
- Сравнит с пилотом
- Соберёт финальный отчёт

После выполнения смотри:
- `analysis/FINDINGS.md` - главный отчёт с выводами
- `analysis/comparison_vs_pilot.md` - сравнение с пилотом
- `analysis/per_cell_summary.csv` - таблица всех ячеек
- `analysis/<cell_id>/biplot.png` - PCA-биплот по каждой ячейке
- `analysis/<cell_id>/corr_heatmap.png` - корреляции конструктов

## Часть 5.7 - практически применимые выходы для 3 аудиторий (~3 минуты)

Это новый и важный шаг. Эксперимент производит данные; рендеры производят то, что можно показать живым людям.

### Шаг 1: синтез операторских инсайтов

```
python operator_synthesis.py
```

Этот шаг **не делает API-вызовов** - он анализирует уже собранные данные и для каждой ячейки производит `operator_outputs/<cell_id>/operator_insight.json`:
- кто рекомендовал что, и какой персональный фрейм
- скрытое разногласие: где агенты пришли к одному выводу разными путями (это главное открытие, которого нет в стандартной агрегации)
- слепые зоны: оси, по которым ни один вариант не выделяется
- риски, поднятые меньшинством голосов
- 5-8 вопросов оператору, которые стоит задать перед принятием решения

### Шаг 2: визуализации с толкованием

```
python visualizations.py
```

Этот шаг **не делает API-вызовов** - он строит графики на основе уже собранных данных. На каждую ячейку получается 4 PNG-картинки + текстовые толкования (как читать график), на весь эксперимент - cross-cell ландшафт:

В `analysis/<cell_id>/`:
- `agreement_network.png` - **сетевой граф согласия**: 5 кружков-агентов соединены линиями. Толстая зелёная линия = агенты рассуждают одинаково; тонкая красная = существенное разногласие на уровне reasoning. Halo вокруг кружка показывает рекомендацию. Это и есть та визуализация, которая сразу показывает "консенсус не такой как кажется".
- `disagreement_heatmap.png` - матрица 5×5 попарных RMSE с цветовой шкалой
- `biplot_annotated.png` - decision-space с **расшифрованными осями** (PC1 = "что-то ↔ что-то", PC2 = "..."), стрелками-конструктами, и подписями вариантов с указанием рекомендации и фрейма автора
- `consensus_vs_reasoning.png` - bar-chart распределения рекомендаций с предупреждением в углу "⚠ Hidden disagreement detected" если RMSE > 0.5
- `*.interpretation.md` - текстовое толкование для каждой картинки (как читать, что это значит, что делать оператору)

В `analysis/cross_cell/`:
- `landscape.png` - все 10 ячеек на одной карте (количество агентов в большинстве × RMSE в рассуждениях), с зонами DANGER (скрытое разногласие) / SAFE (настоящий консенсус) / DISPERSED (явное несогласие)

### Шаг 2.5: интерактивная визуализация

```
python visualizations_interactive.py
```

Главная "вау"-фича для демонстрации аудиториям. На каждую ячейку строится **отдельный HTML-файл** в `analysis/<cell_id>/interactive_network.html`. Открывается в любом современном браузере (Chrome, Firefox, Safari, Edge), работает офлайн после первой загрузки.

Что внутри:
- **Кликабельный граф 5 агентов**. Каждый кружок - агент, цвет внутри = фрейм (Quantitative/Systems/Engineering/Humanist/Contrarian), цвет ободка = вариант рекомендации.
- **Клик по агенту** → правая панель: полный текст ответа агента, его модель, его конструкты (с поясняющими poles), его рекомендация.
- **Клик по линии связи** → правая панель: попарный RMSE, какие именно 3 конструкта дали наибольшее расхождение, как именно каждый агент рейтил элементы по этим конструктам (таблица 1-7 с красной подсветкой больших расхождений), и **автоматический баннер "Hidden disagreement" если оба агента рекомендовали одно но рейтили по-разному**.
- **Hover по агенту/линии** → подсветка соседей, остальное приглушается.
- **Переключение layout**: force-directed / circle / grid - чтобы найти удобный вид.
- **Левая легенда** объясняет цвета и толщину линий.
- **Верхний баннер** даёт исполнительный summary случая (DANGER/MODERATE/GENUINE/SPLIT).

Это и есть тот инструмент, который **отвечает на твою претензию "не хватает понимания осей и размерности"**: тут не нужно гадать что значит PC1, тут можно ткнуть в любой элемент и получить text-объяснение **на каких именно конструктах** произошло расхождение и **какие были рейтинги по 7-бальной шкале**.

Файл self-contained, можно отправить клиенту/коллеге/в Twitter одним attachment'ом. ~40 KB на ячейку.

## Что точно записывается на диск (для пейпера)

Это важно: после прогона есть **полный аудит-трейл** в `logs/api_calls/`. На каждый API-вызов сохраняется отдельный JSON-файл со всем содержимым:
- системный промпт целиком
- пользовательский промпт целиком
- сырой ответ модели целиком (не парсенный)
- usage (токены)
- latency в миллисекундах
- timestamp с точностью до микросекунды
- какая именно модель ответила (primary или fallback)
- сколько попыток понадобилось

При 10 ячейках × 5 моделей × 3 фазы получится **150 файлов аудита** в `logs/api_calls/<cell>/`. Каждый - примерно 5-15 КБ JSON. Всего ~1-2 МБ.

В каждом `results/<cell>/cell.json` дополнительно:
- timestamps начала и конца прогона ячейки
- random_seed, использованный для шафла элементов и распределения триад
- per-model breakdown расходов и токенов
- компактный список `api_calls` с ссылкой на полный аудит-файл

В `results/run_manifest.json` (создаётся один раз в начале прогона):
- SHA-256 хэш скрипта `run_experiment.py`
- SHA-256 хэш `config.yaml`
- snapshot цен и context-length всех 5 моделей **на момент запуска**
- версия Python и всех Python-пакетов (requests, numpy, scikit-learn и т.д.)
- timezone и точное время старта
- полный snapshot конфига

Это то, что ICLR/NeurIPS-ревьюверы спрашивают в первую очередь. С такими артефактами на любой вопрос "что было в промпте M3 в условии P на Task B?" есть ответ из файла, а не из памяти.

## Что делать, если что-то сломалось

### `Set OPENROUTER_API_KEY environment variable`
Не положил ключ в переменную окружения, или открыл новое окно PowerShell после временной установки. Положи заново (см. 2.4).

### `X NOT FOUND in OpenRouter catalog`
Slug модели изменился. Зайди на https://openrouter.ai/models, найди текущий, обнови `config.yaml`.

### `All attempts failed for X`
Конкретная модель отдавала ошибки 3 раза подряд. Часто временно. Проверь `logs/run_*.log`, подожди 5-10 минут, запусти `--resume`.

### `BUDGET HARD STOP at $X`
Сработала защита от перерасхода. Открой `config.yaml`, подними `total_budget_usd:` если доверяешь прогону, потом `--resume`.

### HTTP 429 (rate limit)
OpenRouter троттлит. Подожди 10 минут, потом `--resume`.

### `parsed only N constructs from output`
Модель вернула конструкты в неожиданном формате. Скрипт сохранит то, что смог распарсить. Если очень мало - открой `results/<cell>/cell.json`, посмотри `raw_output_excerpt`, при необходимости можешь руками поправить.
