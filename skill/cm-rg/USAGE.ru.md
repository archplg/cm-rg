# Как пользоваться скиллом cm-rg: пошаговая инструкция

Скилл - это папка с инструкциями и скриптами, которую Claude подхватывает
автоматически, когда задача подходит под его описание. Этот скилл проводит
замер по методике Cross-Model Repertory Grid: модели пишут ответы на задачу,
видят анонимные ответы друг друга, формулируют свои критерии различий
(конструкты) и оценивают всех по всем критериям. На выходе - решётка, метрики
согласия и карта.

## Шаг 1. Установка (один раз)

Выберите любой из трёх способов:

**Способ А - кнопка Save skill (самый простой).** В чате Claude, куда был
отправлен файл `cm-rg.skill`, нажмите на карточке файла кнопку Save skill.
Скилл появится в вашем профиле и будет доступен во всех разговорах. Если
кнопки нет - в вашей организации отключено сохранение скиллов, используйте
способ Б.

**Способ Б - папка Claude Code.** Скопируйте папку `skill/cm-rg` целиком в
`C:\Users\Sergey\.claude\skills\cm-rg` (если папки skills нет - создайте).
Проверка: откройте Claude Code, спросите "какие скиллы тебе доступны" -
в списке должен быть cm-rg.

**Способ В - внутри проекта.** Если работаете в Claude Code в папке
репозитория, скопируйте `skill/cm-rg` в `.claude\skills\cm-rg` в корне
репозитория - скилл будет действовать только в этом проекте.

Для режимов с расчётами нужен Python 3.10+ и пакеты numpy, matplotlib,
requests: `pip install numpy matplotlib requests`.

## Шаг 2. Демо-замер без ключей (15 минут, бесплатно)

Работает в Claude Code и Cowork - там, где Claude умеет запускать субагентов.

1. Напишите Claude: "Запусти мини-замер CM-RG на моделях, которые можешь
   поднять субагентами" (или просто "прогони репертуарную решётку на моделях
   Claude").
2. Claude сам пройдёт 4 фазы: свободные ответы - анонимизация - выявление
   конструктов - перекрёстные оценки, затем посчитает метрики и напишет отчёт.
3. На выходе в рабочей папке: `grid.json` (все данные), `analysis/metrics.json`
   (цифры), два графика (тепловая карта согласия и карта ответов), `REPORT.md`.

Свою задачу вместо встроенной (M&A) можно дать так: "возьми вот этот кейс
как задачу для решётки" + текст кейса на 200-400 слов с 4-5 вариантами решения.

## Шаг 3. Замер любых моделей через OpenRouter

OpenRouter - агрегатор, дающий доступ к сотням моделей (GPT, Gemini, Qwen,
DeepSeek и другим) по одному API-ключу.

1. **Ключ.** Зайдите на openrouter.ai - Sign in - раздел Keys - Create Key.
   Пополните баланс на $5-10 (Credits). Ключ никому не показывайте и не
   вставляйте в файлы и чаты.
2. **Задайте ключ в PowerShell** (действует до закрытия окна):
   ```
   $env:OPENROUTER_API_KEY = "sk-or-v1-ваш-ключ"
   ```
   Навсегда: `setx OPENROUTER_API_KEY "sk-or-v1-ваш-ключ"` (после этого
   открыть новое окно PowerShell).
3. **Выберите модели** на openrouter.ai/models - нужны точные слаги вида
   `openai/gpt-5.5`, `google/gemini-3.1-pro`. От 3 до 12 моделей; интереснее
   всего смешивать лаборатории и ценовые уровни.
4. **Проверка без трат** (из папки скилла):
   ```
   cd C:\Users\Sergey\archipelago_cross_model\skill\cm-rg\scripts
   python run_openrouter.py --selftest
   ```
   Должно закончиться строкой SELFTEST PASSED - значит окружение в порядке.
5. **План и оценка стоимости** (тоже без трат):
   ```
   python run_openrouter.py --models "anthropic/claude-sonnet-4.6,openai/gpt-5.5,google/gemini-3.1-pro" --brief ..\assets\task_brief_ma.md --out run1 --dry-run
   ```
   Скрипт проверит слаги по живому каталогу и покажет число вызовов и
   ориентировочную стоимость.
6. **Запуск** - та же команда без `--dry-run`. Жёсткий потолок расходов по
   умолчанию $5, менять так: `--max-usd 10`. Обычный порядок цен: 3-5 моделей
   на одной задаче - от центов до пары долларов, зависит от дороговизны
   моделей (потолок в любом случае не даст выйти за рамки).
7. **Анализ**:
   ```
   python analyze_grid.py run1\grid.json run1\analysis
   ```
   Цифры - в `run1\analysis\metrics.json`, картинки рядом.

Проще всего: если скилл установлен, просто скажите Claude "запусти CM-RG на
GPT-5.5, Gemini 3.1 Pro и Qwen 3.7 Max через OpenRouter, потолок $5" - он сам
выполнит шаги 4-7 и напишет отчёт (ключ всё равно задаёте вы сами, шаг 2).

## Шаг 4. Только анализ готовых данных

Если решётка уже есть (свой прогон или опубликованный датасет): скажите Claude
"проанализируй grid.json по методике CM-RG" - или запустите `analyze_grid.py`
вручную, как в шаге 3.7.

## Шаг 5. Панель моделей под вашу задачу (исходная идея проекта)

Зачем: перед оркестрацией сложного вопроса понять, какие модели дадут
независимые взгляды, а какие продублируют друг друга - и не платить за три
копии одного мнения.

1. Прогоните решётку на СВОЕЙ задаче: ваш рабочий запрос и есть бриф первой
   фазы. Через субагентов (бесплатно) или через OpenRouter по кандидатам,
   которых рассматриваете для оркестрации. Проще всего сказать Claude:
   "собери панель моделей под эту задачу: <текст задачи>, кандидаты - такие-то".
2. Затем:
   ```
   python compose_panel.py run1\grid.json run1 --k 3
   ```
   Флаг `--must-include слаг` закрепляет в панели модель, от которой вы не
   готовы отказаться.
3. В `PANEL.md` будет: рекомендованная панель максимального разнообразия,
   блоки согласия (r больше 0.7 - фактически один голос), число реальных
   независимых голосов, кого ставить синтезатором (самая "центральная"
   модель), поправки калибровки для усреднения оценок и предупреждение, если
   из этого пула больше разнообразия не выжать - тогда расширяйте пул, а не k.

Как пользоваться панелью: опросить модели независимо (без обмена ответами),
синтез поручить рекомендованной центральной модели; совпадение ответов
считать сильным сигналом только когда оно пересекает границы блоков - согласие
внутри блока это одно мнение, а не несколько. Для быстрых бытовых вопросов
замер не нужен: берите ориентир из атласа исследования - один западный
флагман + один китайский флагман + одна открытая модель.

Честная оговорка: разнообразие панели - измеренный факт; "разнообразная
панель даёт более надёжный ответ" - рабочая гипотеза, подкреплённая
литературой об ансамблях, но не доказанная этим прогоном.

## Как читать результаты

- `mean_pairwise_r` - среднее согласие оценщиков. Ориентиры из исследования:
  0.20 - средний уровень между 36 моделями разных лабораторий; 0.34-0.45 -
  внутри одной семьи; выше 0.7 - модели фактически смотрят одними глазами.
- `calibration_mean_rating` - кто систематически ставит выше/ниже всех.
- Тепловая карта - блоки согласных между собой моделей видны сразу.
- Карта ответов (PCA) - чем дальше точки, тем более разными группа считает
  эти ответы; подписи осей - в `pca_top_loadings`.

## Если что-то пошло не так

- **401 Unauthorized** - ключ не задан или неверен (шаг 3.2).
- **402 / insufficient credits** - пополните баланс OpenRouter.
- **"not in OpenRouter catalog"** - опечатка в слаге; сверьте на
  openrouter.ai/models.
- **"batch unparseable - cells left null"** - модель вернула битый JSON даже
  после повтора; пустые ячейки честно попадут в отчёт как null, число - в
  `metrics.json`. Можно просто перезапустить прогон.
- **STOPPED: spend cap** - упёрлись в потолок; частичные результаты сохранены,
  поднимите `--max-usd` и запустите заново.
- Сырые ответы всех фаз лежат в папке прогона - ничего не теряется.
