# Cross-Model Repertory Grid (CM-RG)

[English](README.md) · Русский

Методика и датасет для измерения оценочного разнообразия фронтирных языковых
моделей на основе психологии личных конструктов Джорджа Келли.

## Что это

CM-RG переносит технику репертуарных решёток (Kelly, 1955) из психологии
человека на языковые модели. Репертуарная решётка - это способ выяснить, какими
собственными критериями человек (здесь - модель) различает объекты, вместо того
чтобы навязывать ему критерии извне. Каждая модель пишет свободный ответ на
задачу-кейс, затем участвует в триадном выявлении конструктов: видит три
анонимных ответа коллег и формулирует биполярные различия ("решительный -
осторожный"). После этого каждая модель оценивает каждый ответ по объединённому
списку конструктов по шкале 1-7. Результат - структурированная решётка,
показывающая, как каждая модель по-своему нарезает оценочное пространство.

В этом репозитории: полный пайплайн, датасет (кодовое имя "Archipelago") и все
анализы к статье.

## Фаза 2L (июнь 2026) - фронтирный масштаб

Крупнейшее на сегодня эмпирическое исследование перекрёстных оценок "все против
всех":

| Метрика | Значение |
|---|---|
| Всего оценок | 3 055 153 |
| Эмерджентных конструктов | 86 418 |
| Пар оценщик x оцениваемый | 13 928 |
| Средняя корреляция между оценщиками (Pearson r) | 0.200 |
| Моделей x провайдеров x ценовых уровней | 36 x 12 x 3 |
| Бюджет эксперимента | $112.89 |

Три главных результата:

1. Фронтирные модели достигают лишь слабого консенсуса (r = 0.200) даже при
   оценке одинакового анонимного контента по одинаковым конструктам. Миф об
   "объективном ИИ-судье" эмпирически опровергнут.
2. Существует "западный кластер консенсуса": флагманы Anthropic, OpenAI,
   Google и xAI согласны между собой втрое сильнее среднего (r = 0.34) -
   след общей школы RLHF-обучения.
3. Эффект калибровки по ценовому уровню: модели среднего уровня систематически
   ставят более высокие оценки (3.84) чем дешёвые (3.57) и флагманы (3.78),
   независимо от того, кого оценивают.

Отчёты: [`REPORT_SCIENTIFIC.md`](REPORT_SCIENTIFIC.md) (академический) ·
[`REPORT_BLOG.md`](REPORT_BLOG.md) (простым языком, по-русски).

## Попробовать за 15 минут - скилл для Claude

В папке [`skill/cm-rg/`](skill/cm-rg/) лежит скилл (набор инструкций и
скриптов, который Claude подхватывает автоматически). Три режима: мини-замер
без API-ключей на субагентах семейства Claude прямо в Claude Code или Cowork;
замер **любых 3-12 моделей из каталога OpenRouter** встроенным раннером
(`scripts/run_openrouter.py` - оценка стоимости до запуска, жёсткий потолок
расходов, оффлайн-самотест); анализ готовых решёток. Пошаговая инструкция:
[`skill/cm-rg/USAGE.ru.md`](skill/cm-rg/USAGE.ru.md).

Пример результата: [`examples/claude-family-mini/`](examples/claude-family-mini/) -
мини-решётка на 4 моделях Claude: все четыре единогласно выбрали один вариант
решения, но малая модель (haiku) применяет конструкты заметно иначе, чем трио
больших (r = 0.31-0.42 против 0.90-0.93 внутри трио). Решение одно - способ
видеть разный.

Скилл замыкает и исходную задачу методики - **сборку панели моделей под
конкретный запрос**: прогоните решётку на своей задаче, и
`scripts/compose_panel.py` порекомендует, какие модели сочетать - панель
максимального разнообразия, блоки согласия ("три западных флагмана - один
голос"), сколько независимых голосов реально есть, кому поручить синтез и
поправки калибровки. Пример: [`examples/claude-family-mini/PANEL.md`](examples/claude-family-mini/PANEL.md).

## Устойчивость к загрязнению обучающих данных

В отличие от бенчмарков с правильными ответами, в CM-RG нечего заучивать:
конструкты рождаются в момент эксперимента, все ответы анонимизируются до
оценивания, а шкала относительная (сравнение ответов между собой, а не с
эталоном).

## Методика одним абзацем

Для каждой ячейки (задача x условие x прогон) модели независимо пишут
рекомендацию в свободной форме. Ответы очищаются от само-идентификации и
перемешиваются под анонимными метками E1..EN. Затем каждая модель по триаде
ответов формулирует биполярные конструкты ("толерантный к риску - избегающий
риска"), и каждая модель оценивает каждый ответ по каждому конструкту (1-7).
Агрегация по ячейкам даёт PCA-разложения, кластеры, прокрустов анализ и
разложение дисперсии - все скрипты в репозитории.
